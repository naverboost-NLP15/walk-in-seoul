## 주제 선정 배경

<h4>외국인·초행 여행자의 어려움</h4>
서울은 풍부한 문화·역사 자원을 갖추고 있지만, 외국인이나 초행 여행자는 도보 여행 중 번역·검색을 반복해야 하는 불편.<br>


<h4>제한된 실시간 상황 반영</h4>
기존 관광 서비스는 대부분 정적 정보 제공에 그쳐 실시간 상황(날씨/혼잡/사용자 위치)을 반영하지 못함.<br>

<br>

## 문제 정의 및 해결 방향

| 문제 정의                                                                                                                                                  | 해결 방향                                                                                                                                                                                  |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **파편화된 정보 탐색과 관광 몰입 저하**<br><br>도보 여행 중, 장소의 의미를 알기 위해 지도 앱, 검색 앱, 번역 앱을 계속 오가며 관광 흐름이 끊김.             | **AI 에이전트 기반 'Push' 정보 제공**<br><br>사용자가 검색하지 않아도, AI 에이전트가 현재 위치의 핵심 스토리를 자동으로 요약/번역 및 음성(TTS)제공.                                        |
| **사실 나열식 안내**<br><br>기존 관광 안내는 "이곳은 OOO입니다"라는 사실 전달에 그쳐, 이 장소가 중요한 이야기를 전달하지 못함.                             | **RAG/LLM 기반 '스토리텔링' 생성**<br><br>신뢰할 수 있는 역사/문화 이야기를 DB를 RAG로 구축. LLM이 사용자의 관심사(e.g., "K-POP", "역사")에 맞춰 사실을 매력적인 이야기로 재가공하여 전달. |
| **정적 정보 제공**<br><br>날씨가 궂거나, 특정 장소가 너무 혼잡하거나, 내가 K-POP 팬임에도 불구하고 모든 사용자에게 동일한 역사 정보만 제공함.              | **동적 정보 제공**<br><br>날씨/혼잡도 API를 도구(Tool)로 활용. Agent가 "지금 비가 오, 500m 내 실내 관광지인 'OO박물관'을 방문드리는 걸 추천합니다."처럼 능동적으로 동선을 수정/제안.       |
| **기계적 번역의 한계와 언어 장벽**<br><br>단순 번역기는 문화적 뉘앙스나 역사적 용어를 어색하게 번역하여 몰입 저해, 또한 영어/일본어 외 다국어 지원이 부족. | **다국어 LLM 및 고품질 TTS 활용**<br><br> 다국어 LLM을 통해 자연스러운 해설 스크립트를 생성. 사용자의 모국어로 유창한 음성 안내(TTS)를 제공하여 '개인 가이드' 경험 구현.                   |

## Key Features
1. 실시간 장소 인식 기반 설명
* 사용자 위치를 기반으로 주변 관광지, 역사적 의미, 장소 맥락을 AI가 자동 안내
* 외국인 사용자도 바로 이해할 수 있도록 다국어 지원 + TTS 제공
2. 개인화된 도보 여행 동선 추천
* 기상청 API → 날씨
* 지도 API → 혼잡도, 거리
* 사용자 선호 → 문화 / 전통 / 힐링 / 쇼핑 등 종합 분석하여 현재 조건에 가장 맞는 관광지와 동선 자동 추천
3. RAG 기반 사실성 강화
* AI가 잘못된 정보를 생성하지 않도록 관광지 정보·역사 데이터 등을 RAG로 제공하여 정확성 확보
4. 능동적 Agent 시스템
* 목적에 따라 계획 수립
* 필요 API 자동 호출
* 사용자 행동에 따라 실시간 코스 재조정

## 사용 기술
* LLM : Upstage Solar Pro 2 API
* Agent : Tool 기반 능동적 계획 및 정보 호출
* RAG : 관광지 정보/지식 제공
* OCR/지도 API : 위치·혼잡도 실시간 확인
* 기상청 API : 시간대별 날씨 확인

<br>

## Service Flow
1. 온보딩
    * 간단한 취향(문화·자연·쇼핑·역사 등) 선택
2. 실시간 추천
    * 위치·날씨·혼잡도·거리 등을 종합해 top-K 관광지 추천
3. 상세 정보 제공
    * 관광지 스토리, 역사적 맥락
    * 왜 추천되었는지 이유 설명
    * 다국어 지원 + 음성 안내(TTS)
4. 능동적 동선 안내
    * 주변 이동 경로 실시간 최적화
    * 위험 구간/혼잡 지역 회피
    * 일정 시간 내 코스 자동 리빌딩

<br>

## Expected Impact (사회·경제적 효과)
1) 이용자 편의성 향상
* 장소마다 숨은 의미를 실시간 안내
* 검색/번역 없이도 여행 몰입도 크게 상승
* 초행자·외국인에게 높은 접근성 제공
2) 지속 가능한 서울 관광에 기여
* 혼잡도 기반 분산 유도 → 특정 지역 과밀 완화
* 골목·비주류 관광지까지 발굴하여 지역 균형 발전 가능
3) 지역 경제 활성화
* 외국인의 소비 동선 개선
* 주변 문화 공간·상권 방문 증가 기대
